## Faz 8 — `extends LogTest` → `static import`

**Hedef:** `LogTest`'i miras yerine import edilen utility'ye çevir. Java tek-kalıtımı serbest kalsın, sınıf hiyerarşisi düzleşsin.

### Önkoşul Doğrulaması
1. `LogTest`'ten miras alan tüm sınıfları listele:
   ```
   Grep: pattern = "extends LogTest", output_mode = files_with_matches
   ```
   Beklenen: `BaseStep`, `TakeScreenShot`, ve test sınıfları.
2. `BaseStep`'ten miras alanları listele:
   ```
   Grep: pattern = "extends BaseStep", output_mode = files_with_matches
   ```
3. **Karar noktası:** `LogTest`'in alt sınıflar tarafından override edildiği bir metod var mı?
   ```
   Grep: pattern = "@Override", glob = "**/*.java", -A = 1
   ```
   Yoksa bu refaktör güvenle yapılabilir.

### Faz 8.1 — `LogTest`'i `final` Yap
1. `LogTest.java` (`:8`):
   ```java
   public final class LogTest {
   ```
2. `@ExtendWith(TestResultLogger.class)` annotation'ını **kaldır** (`:7`) — utility sınıfında olmamalı.
3. Derleme şu an **kırılır** — sonraki fazda düzeltilecek.

### Faz 8.2 — `BaseStep` ve `TakeScreenShot`'tan `extends` Kaldır
1. `BaseStep.java` (`:16`):
   ```java
   // ÖNCE:
   public class BaseStep extends LogTest {
   // SONRA:
   public class BaseStep {
   ```
2. Üst kısma ekle:
   ```java
   import static LogTest.*;
   ```
3. `TakeScreenShot.java` (`:15`) için aynı işlem.
4. Derle, eksik referansları düzelt.

### Faz 8.3 — Test Sınıflarına `@ExtendWith` Ekle
1. `LogTest`'ten miras alan test sınıflarını bul (Faz 8.0'da listelendi).
2. Her birine sınıf seviyesinde ekle:
   ```java
   @ExtendWith(TestResultLogger.class)
   public class DhrAllPageTest extends BaseStep { ... }
   ```
3. `BaseStep extends LogTest` zincirini değiştirdiğin için artık `extends LogTest` bağı yok — testler `LogTest` metodlarını çağırıyorsa `import static LogTest.*` ekle.

### Faz 8.4 — Test Sınıflarında Static Import
1. Her test sınıfı için `info`, `error`, `stepInfo` vb. çağrıları varsa ekle:
   ```java
   import static LogTest.*;
   ```

### Doğrulama
- `mvn clean test` — tüm testler yeşil.
- Log dosyasında format aynı kalmalı (output bozulmadı).
- TestResultLogger callback'leri (PASSED/FAILED) hâlâ çalışmalı.

### Geri Alma
- Branch'i sil. Bu refaktör tek atomik birim olarak revert edilmeli.

## Faz 9 — Implicit Wait'i Kaldır

**Hedef:** Implicit + explicit wait karışımını sonlandır. Sadece explicit wait kalsın.
**En riskli faz** — gizli implicit wait'e bel bağlayan her noktada test yavaşlayabilir/patlayabilir.

### Faz 9.0 — Hazırlık Taraması
1. `BaseStep.driver` üzerinde implicit wait kullanan testleri belirle:
   ```
   Grep: pattern = "implicitlyWait", output_mode = content
   ```
2. `setImplicitWait` çağıranları bul:
   ```
   Grep: pattern = "setImplicitWait\\(", output_mode = content
   ```
3. `findElement` veya `findElements` çağrılarının **wait olmadan** yapıldığı yerleri listele (implicit wait'e gizlice bağımlı):
   ```
   Grep: pattern = "driver\\.findElement", output_mode = content
   ```

### Faz 9.1 — Implicit Wait'i Kapat
1. `BaseStep.openChromeDriver()` (`:46`) içinde:
   ```java
   // SİL:
   driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(DEFAULT_TIMEOUT));
   ```
2. `setImplicitWait` metodunu (`:79-86`) **deprecate et** veya sil:
   ```java
   @Deprecated
   public static void setImplicitWait(int seconds) {
       warn("setImplicitWait is deprecated; use explicit waits");
   }
   ```

### Faz 9.2 — Wait'siz Finder'ları Düzelt
Faz 9.0'da listelenen her `driver.findElement(...)` çağrısı için:
- Doğrudan `findElementXpathWithWait`/`findElementIdWithWait` versiyonlarını kullan.
- `BaseStep` dışındaki kodda direkt `driver.findElement` varsa, `findElement*WithWait` ekvivalentine çevir.

### Faz 9.3 — Test Çalıştırma & Stabilizasyon
1. Tüm test paketini çalıştır.
2. Yeni "Element not found" / "TimeoutException" hatalarını topla.
3. Her biri için seçenek:
   - **Timeout artır** → ilgili çağrıda `TimeOut.LONG` veya özel süre kullan.
   - **ExpectedCondition değiştir** → presence vs visibility vs clickable.
   - **Gerçek bug** → element zaten gelmiyor, test düzeltilmeli.

### Doğrulama
- Tüm testler yeşil.
- Test süresi **artmamalı** (ideal olarak azalmalı — implicit wait gereksiz beklemeleri kaldırır).

### Geri Alma
- Tek satırlık `implicitlyWait` çağrısını geri ekleyerek hızlı kurtarma mümkün.

## Faz 10 — `static driver` → `ThreadLocal<WebDriver>`

**Hedef:** Paralel test çalıştırmaya altyapı hazırlığı. Şu an testler sıralı çalışıyorsa **fonksiyonel kazanç yok**, sadece geleceğe yatırım.

### Önkoşul Karar Noktası
**Bu fazı uygulamadan önce sor:** Paralel test gerçekten ihtiyaç mı?
- Test paketi 5+ dakika sürüyor mu?
- JUnit 5 paralelleştirmesi etkin mi (`junit-platform.properties`)?
- CI'de paralel runner'lar var mı?

Cevap "hayır"sa **bu fazı erteleyin** — refactor maliyeti faydadan büyük.

### Faz 10.1 — `BaseStep.driver`'ı ThreadLocal'a Çevir
1. `BaseStep.java` (`:19`):
   ```java
   // ÖNCE:
   public static WebDriver driver;
   // SONRA:
   private static final ThreadLocal<WebDriver> driverHolder = new ThreadLocal<>();

   public static WebDriver getDriver() {
       return driverHolder.get();
   }

   public static void setDriver(WebDriver d) {
       driverHolder.set(d);
   }
   ```
2. **Geriye dönük uyumluluk:** `public static WebDriver driver`'a doğrudan erişen tüm kod kırılır. Şu çağrıları taramak ZORUNLU:
   ```
   Grep: pattern = "BaseStep\\.driver", output_mode = content
   Grep: pattern = "\\bdriver\\b" (BaseStep'in iç metodları için), output_mode = content
   ```
3. Her `driver` referansını `getDriver()`'a çevir.

### Faz 10.2 — `openChromeDriver` ve `driverQuit` Güncellemesi
1. `openChromeDriver` (`:27`):
   ```java
   ChromeDriver d = new ChromeDriver(options);
   setDriver(d);
   wait = new WebDriverWait(d, Duration.ofSeconds(DEFAULT_TIMEOUT));
   ```
2. `wait` da `ThreadLocal` olmalı — yoksa thread'ler arası paylaşılır.
3. `driverQuit` (`:62`):
   ```java
   WebDriver d = getDriver();
   if (d != null) {
       d.quit();
       driverHolder.remove(); // ÖNEMLİ: thread leak önler
   }
   ```

### Faz 10.3 — `TestResultLogger.getDriverFromTestInstance` Güncellemesi
Plan 1-4 Faz 3'te `BaseStep.driver` direkt erişimi yapılmıştı. Şimdi:
```java
private WebDriver getDriverFromTestInstance(String testName) {
    return BaseStep.getDriver();
}
```

### Faz 10.4 — Paralel Çalıştırma Konfigürasyonu (Opsiyonel)
1. `src/test/resources/junit-platform.properties` oluştur veya güncelle:
   ```properties
   junit.jupiter.execution.parallel.enabled = true
   junit.jupiter.execution.parallel.mode.default = concurrent
   ```
2. Test sınıfları arasında paylaşılan state olup olmadığını gözden geçir.

### Doğrulama
- **Sıralı modda** tüm testler yeşil olmalı (refactor regresyon yaratmamalı).
- **Paralel modda** (Faz 10.4 yapıldıysa) en az 2 test paralel çalışırken birbirine karışmamalı.
- Driver leak yok: test sonunda `driverHolder.remove()` çağrılıyor mu?

### Geri Alma
- Branch'i sil. Bu refactor tek atomik birim.
