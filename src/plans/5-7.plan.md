### Adımlar
1. `src/test/java/BaseStep.java` (`:137-147`) `clickElement` metodunu güncelle:
   ```java
   public static void clickElement(WebElement element, String elementDescription) {
       try {
           actionInfo("Click", elementDescription);
           WebDriverWait localWait = new WebDriverWait(driver, Duration.ofSeconds(DEFAULT_TIMEOUT));
           localWait.until(ExpectedConditions.elementToBeClickable(element));
           element.click();
           stepInfo("Element clicked successfully: " + elementDescription);
       } catch (Exception e) {
           logException("Click operation", e);
           TakeScreenShot.takeFailureScreenshot("clickElement", driver, "Failed to click: " + elementDescription);

       }
   }
   ```

## Faz 3 — Finder Metodlarını Tutarlılaştır (null → exception)

**ÖNKOŞUL:** Faz 0.3 raporu — null-bağımlı her çağrı önce `try/catch` ile sarılmış olmalı (alt-faz 3.0).

### Faz 3.0 — Çağıranları Defansif Hale Getir
Faz 0.3'te bulunan her "null-check yapan" çağrı için:
- **Eğer optional davranışsa** (örn. `legitimateChecks` içindeki promo banner): `isElementPresent(By.id(...))` ile değiştir.
- **Eğer "var ama bulunmadı, devam et" davranışsa**: `try { ... } catch (NoSuchElementException e) { ... }` ile sar.

`BaseStep.legitimateChecks()` (`:460-467`) için örnek:
```java
public void legitimateChecks() {
    if (isElementPresent(By.id("promo-banner"))) {
        WebElement banner = findElementIdWithWait("promo-banner", TimeOut.SHORT.value);
        clickElement(banner, "Promo banner");
    }
}
```

### Faz 3.1 — Finder İmzalarını Birleştir
Aşağıdaki metodları, başarısızlıkta `NoSuchElementException` fırlatacak şekilde güncelle:
- `findElementXpath` (`:163`)
- `findElementIdWithWait` (`:218`)
- `findElementClassNameWithWait` (`:234`)
- `findElementCssSelectorWithWait` (`:250`)
- `findElementNameWithWait` (`:266`)
- `findElementLinkTextWithWait` (`:282`)
- `findElementTagNameWithWait` (`:298`)

Şablon (`findElementXpathWithWait` zaten doğru kalıbı kullanıyor — örnek alın):
```java
public static WebElement findElementIdWithWait(String id, int timeoutSeconds) {
    try {
        WebDriverWait customWait = new WebDriverWait(driver, Duration.ofSeconds(timeoutSeconds));
        WebElement element = customWait.until(ExpectedConditions.presenceOfElementLocated(By.id(id)));
        debug("Element found by id with wait: " + id);
        return element;
    } catch (TimeoutException e) {
        error("Element not found by id within " + timeoutSeconds + " seconds: " + id);
        TakeScreenShot.takeFailureScreenshot("findElementIdWithWait", driver, "Element not found: " + id);
        throw new NoSuchElementException("Element not found after " + timeoutSeconds + " seconds: " + id, e);
    }
}
```

### Faz 3.2 — `findElementXpathWithWait`'in Condition Tutarsızlığını Çöz
Şu an `elementToBeClickable` kullanıyor (`:194`), diğerleri `presenceOfElementLocated`. **Karar gerekli:**
- Seçenek A — Hepsi `presenceOfElementLocated` (DOM'da var). Click sorumluluğu `clickElement`'in waiti'ne devredilir (Faz 2).
- Seçenek B — Hepsi `elementToBeClickable`. Daha katı ama input-only finder'lar için fazla.

**Öneri: Seçenek A.** Tipi tutarlı yapar, `clickElement` zaten clickable bekliyor (Faz 2 sonrası).
