## Faz 2 — `TimeOut.toDuration()` Yardımcı Metodu ✅

**Hedef:** Çağıranlarda tekrar eden `Duration.ofSeconds(TimeOut.X.value)` kalıbını kısaltmak. Mevcut `value` alanı **silinmeyecek** — geriye dönük uyumluluk için kalır.

### Adımlar
1. `src/test/java/TimeOut.java` içine ekle:
   ```java
   import java.time.Duration;

   public Duration toDuration() {
       return Duration.ofSeconds(value);
   }
   ```
2. **Çağıranları bu fazda güncelleme** — sadece metodu eklemek yeterli. Çağıran refaktörü ayrı (Faz 5+) bir iş; bu fazda davranış değişmez.

### Doğrulama
- Derleme hatasız.
- `TimeOut.SHORT.value` hâlâ çalışıyor (test edilmiş bir test sınıfını çalıştır).

### Geri Alma
- Eklenen 4 satırı sil.

## Faz 3 — `TestResultLogger`'da Reflection'ı Kaldır ✅

**Hedef:** `getDriverFromTestInstance` içindeki kırılgan reflection'ı, `BaseStep.driver`'a doğrudan statik erişimle değiştirmek. Davranış aynı kalır çünkü field zaten `public static`.

### Önkoşul Doğrulaması
1. `BaseStep.driver` alanının `public static WebDriver` olduğunu teyit et (`BaseStep.java:19`).
2. Başka hiçbir yerde aynı isimde driver field'ı tanımlanmadığından emin ol (Grep ile).

### Adımlar
1. `src/test/java/TestResultLogger.java` içinde:
   - `import java.lang.reflect.Field;` satırını sil.
   - `getDriverFromTestInstance(String testName)` metodunu şununla değiştir:
     ```java
     private WebDriver getDriverFromTestInstance(String testName) {
         return BaseStep.driver;
     }
     ```
   - `testInstances` map'i artık sadece bu metod tarafından kullanılıyorsa, `recordTestStart` ve `cleanupTestData` içindeki `testInstances.put/remove` çağrıları çalışmaya devam etsin (ileride 8-10 planında temizlenecek). Bu fazda **dokunma**.

### Doğrulama
- Bir testi kasten başarısız yap (`assert false` ekle) → failure screenshot'ının hâlâ alındığını teyit et.
- Test'i geri al, başarı screenshot'ının hâlâ alındığını teyit et.

### Geri Alma
- `git checkout -- src/test/java/TestResultLogger.java`

## Faz 4 — Exception Sarma Kalıbını Düzelt ✅

**Hedef:** `new Exception(cause)` ile orijinal exception sarılıyor → stack trace bozuluyor. Log4j'in native cause-aware imzasını kullan.

### Adımlar
1. `src/test/java/TestResultLogger.java` içinde:
   - **Satır ~76** (`testAborted` metodu içinde):
     ```java
     // ÖNCE:
     LogTest.logException("Test abortion", new Exception(cause));
     // SONRA:
     logger.error("Test abortion", cause);
     ```
   - **Satır ~106** (`testFailed` metodu içinde):
     ```java
     // ÖNCE:
     LogTest.logException("Test failure", new Exception(cause));
     // SONRA:
     logger.error("Test failure", cause);
     ```
2. **Not:** `cause` zaten `Throwable` — `Exception`'a wrap etmek tip uyumsuzluğu yaratıyor olabilir. `logger.error(String, Throwable)` overload'u Throwable kabul ettiği için sorun yok.

### Doğrulama
- Bir testi kasten başarısız yap → log dosyasında stack trace'in **gerçek hata kaynağına** kadar gittiğini gör (önceden `new Exception(...)` satırından kesiliyordu).

### Geri Alma
- `git diff` ile değişiklikleri görüntüleyip seçici geri alma.
